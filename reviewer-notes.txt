XOR-ENCODED CONFIGURATION VALUES
---------------------------------
The Vite plugin at plugins/obscureEnvKeys.ts XOR-encodes three build-time
configuration values before they are written into the bundle:
  - VITE_API_KEY            (shared secret for Vercel proxy)
  - VITE_SPOTIFY_CLIENT_ID  (Spotify PKCE client — public by Spotify's model)
  - VITE_GOOGLE_DESKTOP_CLIENT_ID (Google Desktop-app client)

Why: Automated string-pattern scanners flag alphanumeric 32-char strings as
potential leaked secrets. XOR byte arrays do not match those patterns.

What this is NOT:
  - Does not obfuscate code logic
  - Decoding routine is a readable one-liner IIFE
  - No encrypted or hidden behavior

Decoding example:
  (()=>{ const _b=[...bytes], _s=seed; return _b.map((b,i) =>
    String.fromCharCode(b^((_s*17+i*7)%256))).join('') })()

Source: plugins/obscureEnvKeys.ts


OAUTH FLOWS
-----------

1. Google (Firefox)
   ----------------
   PKCE via chrome.identity.launchWebAuthFlow. Authorization code exchanged
   server-side through Vercel endpoint:
     https://undistractedme.sarojbelbase.com.np/api/auth/google/token

   Bundle contains: VITE_GOOGLE_DESKTOP_CLIENT_ID (XOR-encoded, public)
   Server holds: GOOGLE_CLIENT_SECRET (environment variable only)
   Neither initial exchange nor refresh calls Google directly from extension.
   Source: src/utilities/googleAuth.js (getTokenFirefox, getValidStoredFFToken)


PERMISSIONS & PURPOSE
---------------------
  identity         → Google OAuth (PKCE) + Spotify OAuth
  storage          → Widget settings, OAuth tokens (chrome.storage.local for sensitive)
  geolocation      → Weather widget (user browser prompt required)
  topSites         → Quick Access widget
  tabs             → Media controls routing, look-away notifications
  scripting        → Inject media.js into YouTube/SoundCloud
  notifications    → Event reminders, Pomodoro, look-away breaks
  alarms           → Periodic tasks (event checks, timers)


CONTENT SCRIPTS
---------------
  Sites: YouTube, SoundCloud (media.js)
  Purpose: Poll mediaSession, route MEDIA_ACTION commands [play, pause, next, previous]
  Validation: All messages validated against explicit allowlist
  No: innerHTML, eval, external data exfiltration
